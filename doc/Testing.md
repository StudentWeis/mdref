# mdref 测试体系

本文档描述 `mdref` 项目的测试体系结构、覆盖范围和运行方式。

## 1. 测试分层

项目的测试体系分为四个层次：

```text
┌─────────────────────────────────────────────┐
│          CLI 集成测试 (cli_tests.rs)         │  端到端验证
├─────────────────────────────────────────────┤
│       库 API 集成测试 (lib_*_tests.rs)       │  公开 API 验证
├─────────────────────────────────────────────┤
│       错误路径测试 (error_tests.rs)          │  异常行为验证
├─────────────────────────────────────────────┤
│     核心单元测试 (core/find.rs, core/mv.rs)  │  内部函数验证
├─────────────────────────────────────────────┤
│          性能基准测试 (benches/)              │  吞吐量验证
└─────────────────────────────────────────────┘
```

## 2. 测试文件清单

### 2.1 核心单元测试

位于 `src/core/` 各模块内部的 `#[cfg(test)] mod tests` 中。

**`src/core/find.rs`** — 查找子系统内部函数测试：

| 测试组 | 覆盖内容 |
|---|---|
| `resolve_link` | 相对路径解析、绝对路径、不存在的文件、无父目录、嵌套相对路径 |
| `match_link_to_target` | 文件匹配、文件不匹配、目录包含、目录外部 |
| `process_link` | 无目标时全部接受、文件名不匹配提前返回、匹配链接、不可解析链接 |
| `process_md_file` | 无链接、收集全部链接、行号验证、同行多链接、图片链接、外部 URL、混合内容 |

**`src/core/mv.rs`** — 迁移子系统内部函数测试：

| 测试组 | 覆盖内容 |
|---|---|
| `relative_path` | 同目录、父目录、子目录、深层跨目录、不存在的目标 |
| `replace_link_in_file` | 基本替换、保留其他内容、行号越界、子目录路径替换 |
| `update_reference` | 同目录引用更新、跨目录引用更新 |

### 2.2 库 API 集成测试

位于 `tests/` 目录，通过公开的库 API（`find_links`、`find_references`、`mv_file`）进行测试。

**`tests/lib_find_tests.rs`** — 查找功能测试：

| 测试组 | 覆盖内容 |
|---|---|
| `find_links` 基础 | 基本查找、非 md 文件、链接计数、内容验证、行号验证、空文件、不存在的文件 |
| 图片链接 | 图片链接识别、`./` 前缀图片链接 |
| 同行多链接 | 同一行多个链接的提取和列号递增验证 |
| 嵌套目录引用 | 深层嵌套目录中的引用查找 |
| 自引用 | 文件引用自身的场景 |
| 外部 URL | 外部 URL 的收集和过滤（不匹配为文件引用） |
| `./` 前缀链接 | 使用 `./` 前缀的相对路径链接 |
| 目录目标 | `find_references` 传入目录路径 |
| `Reference` 结构体 | 构造函数和 `Display` trait |

**`tests/lib_mv_tests.rs`** — 移动功能测试：

| 测试组 | 覆盖内容 |
|---|---|
| 基础移动 | 基本移动、内容保留、不存在的源文件、父目录自动创建 |
| 引用更新 | 单引用更新、多引用更新、同名不同目录 |
| 子目录移动 | 移动到子目录、从子目录移动到根目录 |
| 内部链接 | 内部链接更新、图片链接更新 |
| 深层嵌套 | 多层目录间的移动和路径计算 |
| 同文件多行引用 | 同一文件中多行引用同一目标 |
| 自引用 | 被移动文件的自引用更新 |
| 同目录移动 | 等价于 rename 的行为 |
| 与 find 集成 | 移动前后的引用查找一致性 |

**`tests/lib_rename_tests.rs`** — 重命名功能测试：

| 测试组 | 覆盖内容 |
|---|---|
| 基础重命名 | 基本重命名、内容保留 |
| 外部引用更新 | 单引用更新、多引用更新（含跨目录） |
| 内部链接 | 自引用更新、对其他文件链接的保持 |
| 子目录重命名 | 在嵌套目录中重命名 |
| 错误场景 | 不存在的文件 |

### 2.3 错误路径测试

**`tests/error_tests.rs`** — 异常行为和错误处理测试：

| 测试组 | 覆盖内容 |
|---|---|
| IO 错误 | `find_links`、`find_references`、`mv_file` 对不存在文件的错误 |
| 目标覆盖 | `mv_file` 目标已存在时的覆盖行为 |
| 错误展示 | `MdrefError` 三种变体的 `Display` 输出 |
| 边界场景 | 空根目录、非 md 文件、无引用移动、无内部链接移动 |

### 2.4 CLI 集成测试

**`tests/cli_tests.rs`** — 通过 `std::process::Command` 调用编译后的二进制进行端到端测试：

| 测试组 | 覆盖内容 |
|---|---|
| `find` 命令 | 基本查找、不存在的文件、无引用场景 |
| `mv` 命令 | 基本移动、不存在的源文件 |
| `rename` 命令 | 基本重命名、不存在的源文件 |
| 帮助和版本 | `--version`、`--help`、无参数 |
| 端到端 | `mv` 引用更新、`rename` 引用更新 |

### 2.5 性能基准测试

**`benches/benchmark.rs`** — 使用 `criterion` 框架：

- 基于 `benches/mock_generator.rs` 生成的 mock 数据集测试 `find_references` 的扫描性能。
- 模拟大规模目录和文件结构，验证并行扫描的吞吐量。

## 3. 测试环境管理

### 临时目录策略

集成测试和单元测试中涉及文件系统操作的测试，统一采用以下模式：

1. **创建**：每个测试创建独立的临时目录（以 `test_{模块}_{场景}` 命名），避免测试间干扰。
2. **清理**：测试结束后调用 `teardown_test_dir` / `cleanup_test_env` 删除临时目录。
3. **隔离**：每个测试使用唯一的目录名，支持并行执行。

### 测试数据

- `examples/` 目录包含预置的 Markdown 文件结构，供 `lib_find_tests.rs` 中的部分测试使用。
- 其他测试均动态创建测试文件，不依赖预置数据。

## 4. 运行测试

```sh
# 运行全部测试
cargo test

# 运行特定测试文件
cargo test --test lib_find_tests
cargo test --test lib_mv_tests
cargo test --test lib_rename_tests
cargo test --test error_tests
cargo test --test cli_tests

# 仅运行单元测试
cargo test --lib

# 运行性能基准测试
cargo bench
```

## 5. 已知限制

以下场景是当前实现的已知限制，测试中已体现：

- **外部 URL 与 `mv_file`**：`mv_file` 在更新被移动文件的内部链接时，会尝试对所有链接做 `canonicalize`，外部 URL（如 `https://...`）会导致 IO 错误。当前实现不支持在包含外部 URL 的文件上执行 `mv_file`。
- **同一文件多次写回**：同一个文件中有多处引用时，会被多次读写，而非批量更新。
- **非事务性迁移**：迁移过程中如果中途失败，可能出现部分文件已更新的不一致状态。
